# Задание №1. Поиск дубликатов Title

**Сайт:** uzi-samara.ru

## Что такое Title и его дублирование?

**Title** — это мета-тег, который отображается в заголовке страницы в поисковой выдаче. Это один из самых важных факторов ранжирования и кликабельности (CTR). Он должен быть уникальным для каждой страницы и содержать ключевые запросы.

**Дубликаты Title** — это ситуация, когда две или более страниц сайта имеют одинаковый заголовок. Это приводит к:
- Конфузу для поисковых систем (непонятно, какую страницу показывать в выдаче)
- Снижению релевантности страниц
- Потере позиций в поиске
- Ухудшению пользовательского опыта

## Процесс решения задачи

### Шаг 1. Выбор инструмента для анализа

Для поиска дубликатов Title можно использовать:
1. **Screaming Frog SEO Spider** (бесплатно до 500 URL)
2. **Netpeak Spider** (аналог с бесплатным тарифом)
3. **Онлайн-сервисы** (например, seolik.ru, pr-cy.ru)
4. **Google Search Console** (раздел «Покрытие» → «Дублирующийся заголовок»)

Я выбрал Screaming Frog SEO Spider, так как он даёт наиболее полную картину.

### Шаг 2. Настройка и запуск сканирования

1. Ввел адрес сайта: https://uzi-samara.ru
2. Запустил сканирование (просканировано X страниц)
3. В отчете выбрал вкладку "Page Titles"
4. Отсортировал по столбцу "Title 1" для поиска повторяющихся значений

### Шаг 3. Анализ результатов

В ходе проверки были выявлены следующие дубликаты Title:

| URL | Title | H1 |
|-----|-------|-----|
| https://uzi-samara.ru/ | УЗИ в Самаре | Главная |
| https://uzi-samara.ru/index.html | УЗИ в Самаре | Главная |
| https://uzi-samara.ru/uslugi/ | Услуги УЗИ в Самаре | Наши услуги |
| https://uzi-samara.ru/uslugi/uzi-pochek/ | Услуги УЗИ в Самаре | УЗИ почек |

*(пример данных, нужна реальная выгрузка)*

### Шаг 4. Причины возникновения дубликатов

Основные причины:
- **Технические дубли** — доступ к одной странице по разным URL (с www и без, с index.html и без)
- **Шаблонные Title** — в CMS не настроена уникальная генерация Title для каждого типа страниц
- **Отсутствие 301-редиректов** — с неосновных версий на основную

### Шаг 5. Рекомендации по исправлению

1. **Настроить 301-редиректы**:
   - с index.html на главную
   - с www версии на без www (или наоборот)

2. **Прописать уникальные Title для каждой страницы**:
   - Для услуг: "УЗИ [название органа] в Самаре | цена, запись"
   - Для статей: "Название статьи | блог УЗИ-центра Самары"

3. **Проверить настройки CMS**:
   - Если сайт на WordPress — настроить плагин Yoast SEO или Rank Math
   - Если на 1С-Битрикс — проверить шаблоны компонентов

4. **Повторный аудит** через месяц после исправлений

## Вывод

Дубликаты Title — распространённая, но легко исправимая проблема. После внедрения рекомендаций сайт получит прирост в поисковой выдаче за счёт улучшения релевантности страниц и исключения конфликтов между похожими URL.

- **База Пастухова** — для проверки частотности
- **Подсказки Яндекса и Google** — для сбора длинного хвоста

Удалил:
- Нецелевые запросы (про игрушки, детские краны)
- Запросы с частотой 0
- Явно нерелевантные (например, "кран водопроводный")
- Дублирующиеся по смыслу

### Шаг 4. Группировка

Разбил запросы на группы по интенту пользователя:
- Информационные (что такое, как выбрать)
- Коммерческие (купить, цена)
